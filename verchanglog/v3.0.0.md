## 📋 版本概述

v3.0.0 是一次围绕账号安全、后台任务和图床能力的重大升级。版本新增邮箱验证、配置可用性测试、转换任务队列、Redis worker、历史记录详情与差异预览，并扩展百度云 BOS、华为云 OBS、又拍云 USS、MinIO 等图床适配；本地上传也升级为后台任务模式，刷新页面后可恢复进度，适合更长耗时和更大批量的文档迁移场景。

---

## ✨ 新增特性

- 新增邮箱验证能力：注册后发送验证邮件，支持验证链接和重发验证邮件。
- 新增邮箱可达性预检查，注册、修改邮箱和重发验证邮件前会校验邮箱格式与域名可达性。
- 新增配置可用性测试，图床配置保存前和保存后均可触发测试上传。
- 新增持久化转换任务队列，批量转换会创建后台任务并轮询任务进度。
- 新增 Redis 队列支持，启用 Redis 后由后台 worker 消费转换任务；未启用时回退为本进程内存队列。
- 新增转换结果预览/差异对比，转换后可查看前后变化摘要。
- 新增历史记录详情页，可查看替换明细、错误信息并下载转换结果。
- 新增百度云 BOS、华为云 OBS、又拍云 USS、MinIO 图床适配。
- 新增图床来源识别，支持识别百度云 BOS、华为云 OBS、又拍云、MinIO 图片地址。
- 新增 `/api/convert/local-tasks` 接口，本地图片上传替换可创建后台任务并加入队列。
- 本地上传支持刷新后恢复后台任务进度，任务结果自动回填到文档列表。

## 🐛 Bug 修复

- 未验证邮箱的用户找回密码时会明确提示“请完成邮箱验证，启用密码找回”。
- 注册成功后如果邮箱未验证，也会通过右上角 toast 提醒完成邮箱验证以启用密码找回。
- 批量转换遇到本地路径图片时不再直接中断整批任务，会跳过该图片并记录失败详情。
- 优化任务结果回填逻辑，优先按文件名匹配记录，降低同批次多文档结果错配风险。
- 任务详情接口预加载替换明细，确保前端恢复任务后能展示完整记录详情。
- MinIO 的“是否使用 SSL”配置从自由输入改为统一样式下拉框，默认值为 `false`。

## 🚀 性能优化

- 批量转换由后台任务逐文档执行，前端只负责创建任务和轮询进度，减少长耗时请求对页面的阻塞。
- Redis worker 支持配置并发数，便于生产环境按服务器资源调整后台处理能力。
- 本地上传任务会复用上传缓存，同一批任务中相同本地图片避免重复上传。
- 本地上传任务完成或失败后自动清理后端临时文件目录，降低磁盘占用风险。

## 🔧 重构优化

- 后端抽出用户级转换执行逻辑，后台 worker 与同步接口复用同一套转换流程。
- 本地上传清单校验逻辑集中复用，同步接口和后台任务接口保持一致。
- 转换任务模型支持保存任务类型、载荷、进度、成功数、失败数、错误信息和结束时间。
- 历史记录新增替换明细模型，用于记录每张图片的原地址、目标地址、状态和错误原因。
- 前端批量转换和本地上传统一接入任务进度弹窗，任务完成后刷新历史记录。
- 图床配置表单扩展多云字段校验和中文展示，MinIO SSL 使用与图床类型一致的自定义下拉样式。
- 本地上传目标配置下拉框改为向上展开，避免遮挡下方操作区域。

## 🔄 API 变更

新增接口：

- `GET /api/picbed/types`：支持返回新增图床类型与配置字段。
- `POST /api/picbed/configs/test`：测试未保存的图床配置。
- `POST /api/picbed/configs/:id/test`：测试已保存的图床配置。
- `POST /api/auth/email/verify`：验证邮箱。
- `POST /api/auth/email/verification`：重发邮箱验证邮件。
- `POST /api/convert/tasks`：创建批量转换后台任务。
- `GET /api/convert/tasks`：获取转换任务列表。
- `GET /api/convert/tasks/:id`：获取转换任务详情和关联历史记录。
- `GET /api/convert/records/:id`：获取转换历史详情与替换明细。
- `POST /api/convert/local-tasks`：创建本地图片上传替换后台任务。

接口行为调整：

- `POST /api/convert/batch` 的批量转换流程升级为后台任务模式。
- `POST /api/convert/local-batch` 保留同步处理能力，新增的 `/api/convert/local-tasks` 用于后台队列处理。
- `/api/convert/tasks/:id` 返回的记录会预加载 `details` 替换明细。
- 忘记密码接口要求邮箱已验证，未验证时返回 403。

新增图床配置项：

- 百度云 BOS：`access_key_id`、`secret_access_key`、`bucket`、`region`，可选 `storage_path`、`custom_domain`。
- 华为云 OBS：`access_key_id`、`secret_access_key`、`bucket`、`region`，可选 `storage_path`、`custom_domain`。
- 又拍云 USS：`bucket`（服务名）、`operator`、`password`、`custom_domain`，可选 `storage_path`。
- MinIO：`endpoint`、`access_key`、`secret_key`、`bucket`，可选 `region`、`use_ssl`、`storage_path`、`custom_domain`；`use_ssl` 默认 `false`。

## 📊 数据库变更

新增或扩展的数据结构：

- `email_verification_tokens`：保存邮箱验证令牌哈希、用户 ID、过期时间、使用时间、请求 IP 和创建时间。
- `conversion_tasks`：保存转换任务类型、状态、总数、成功数、失败数、消息、错误、任务载荷和时间戳。
- `conversion_record_details`：保存单张图片替换明细，包括原地址、目标地址、状态和错误原因。
- `users.email_verified_at`：记录用户邮箱验证时间。
- `conversion_records.task_id`：关联转换任务与历史记录。
- `conversion_records.converted_content`：保存转换后的 Markdown 内容，支持历史详情下载。

## 🔄 升级说明

从 v2.1.0 升级到 v3.0.0 时，应用启动会通过 GORM AutoMigrate 自动创建或更新邮箱验证、转换任务和替换明细相关表结构。

如需手动执行 SQL，请参考 `backend/migrations/001_init_schema.sql` 中的以下结构：

- `email_verification_tokens`
- `conversion_tasks`
- `conversion_record_details`
- `users.email_verified_at`
- `conversion_records.task_id`
- `conversion_records.converted_content`

建议升级前备份数据库。启用 Redis 后请补充以下环境变量：

```env
REDIS_ENABLED=false
REDIS_ADDR=localhost:6379
REDIS_PASSWORD=
REDIS_DB=0
REDIS_CONVERT_QUEUE=picbed:convert_tasks
CONVERT_WORKER_CONCURRENCY=1
```

启用邮箱验证和密码找回前，请确认 SMTP 与应用基础地址配置正确：

```env
APP_BASE_URL=https://your-domain.com
PASSWORD_RESET_TOKEN_TTL_MINUTES=5
EMAIL_VERIFICATION_TOKEN_TTL_MINUTES=5
SMTP_HOST=smtp.example.com
SMTP_PORT=587
SMTP_SECURITY=auto
SMTP_USERNAME=noreply@example.com
SMTP_PASSWORD=your_smtp_password
SMTP_FROM=noreply@example.com
SMTP_FROM_NAME=PicBed Switcher
```

`SMTP_SECURITY` 可选值为 `auto`、`ssl`、`starttls`、`none`。465 端口通常使用 `ssl`，587 端口通常使用 `starttls`。
