# picbed-switcher v3.0.1 版本更新日志

**发布日期**: 2026-05-19<br>
**提交数量**: 2 次提交<br>
**版本类型**: 修复与体验优化版本

---

## 📋 版本概述

v3.0.1 聚焦 Docker 部署日志体验和转换历史管理能力。该版本消除了容器环境中 `.env` 文件不存在时的无影响警告，补充生产部署示例中的 `GIN_MODE=release`；同时历史记录页面新增多选、全选和批量删除能力，删除历史时会同步清理对应替换明细数据。

---

## ✨ 新增特性

- 历史记录列表新增选择列，可逐条勾选需要删除的转换记录。
- 历史记录表头新增全选复选框，支持一键全选或取消全选当前列表记录。
- 新增历史记录批量删除按钮，未勾选记录时保持禁用，勾选后可执行删除。
- 新增 `DELETE /api/convert/records` 接口，支持按记录 ID 批量删除当前用户的转换历史。

---

## 🐛 Bug 修复

- 修复 Docker 容器中通过环境变量运行时，后端因 `/app/.env` 不存在而输出无影响 warning 的问题。
- 修复历史记录删除方法未注入前端 workspace context，导致点击删除按钮报 `deleteRecords is not a function` 的问题。
- 删除历史记录时会在事务内先删除对应替换明细，再删除历史记录，避免详情数据残留。

---

## 🚀 性能优化

- 历史记录表格超过 15 行后启用内部滚动，并隐藏滚动条，减少页面整体滚动干扰。
- 删除历史记录前会去重和过滤无效 ID，单次最多处理 50 条记录，避免误提交过大请求。

---

## 🔧 重构优化

- `.env` 加载逻辑调整为：文件不存在时静默跳过，仅在文件存在但读取或解析失败时输出 warning。
- 历史记录删除操作使用局部删除状态，不再受其它全局加载状态影响。
- 历史记录列表选中状态会随记录列表刷新自动清理无效 ID。

## 🔄 API 变更

新增接口：

- `DELETE /api/convert/records`：批量删除转换历史。

请求体示例：

```json
{
  "ids": [1, 2, 3]
}
```

接口说明：

- 仅允许删除当前登录用户自己的转换历史。
- 后端会对 ID 去重并过滤无效 ID。
- 单次最多删除 50 条转换历史。
- 删除历史记录时会同步删除 `conversion_record_details` 中的替换明细。

---

## 📊 数据库变更

本版本无新增表结构或字段。

数据删除行为调整：

- 删除转换历史时同步删除对应 `conversion_record_details` 明细记录。

## 🔄 升级说明

从 v3.0.0 升级到 v3.0.1 无需执行数据库迁移。

Docker 或 Docker Compose 部署建议在环境变量中显式设置：

```env
GIN_MODE=release
```

容器内没有 `/app/.env` 文件时，后端将不再输出 `failed to load .env` warning；已有通过 `env_file` 或容器环境变量注入的配置方式保持不变。
