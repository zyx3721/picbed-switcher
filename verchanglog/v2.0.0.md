## 📋 版本概述

v2.0.0 聚焦 Markdown 本地图片迁移与批量任务执行体验升级。新增“本地上传”菜单，可将 Markdown 中引用的本地图片文件上传到目标图床并自动替换为远程地址；同时新增全局处理中弹窗，让批量转换和本地上传替换过程具备明确的执行状态、进度提示和成功失败统计。

---

## ✨ 新增特性

- 新增“本地上传”菜单页，支持上传 Markdown 文档、本地图片文件或图片目录。
- 新增 Markdown 本地图片路径匹配能力，支持 `./images/a.png`、`images/a.png`、Windows 绝对路径和 `<img src="...">` 等引用形式。
- 新增本地图片上传替换流程，将已匹配的本地图片上传到目标图床后回写 Markdown 内容。
- 新增 `/api/convert/local-batch` 接口，支持通过 multipart/form-data 提交 Markdown、本地图片文件和路径映射清单。
- 新增本地图片文件校验与构造能力，限制单张图片大小并校验 MIME 类型。
- 新增全局处理中弹窗，批量任务执行时显示当前第 N / 总数 个文档、成功数、失败数和请勿关闭或刷新提示。
- 批量转换改为逐文档执行，进度弹窗可真实反映当前处理位置。
- 本地上传替换改为逐文档执行，进度弹窗可显示当前上传替换文档及匹配图片数量。

## 🚀 性能优化

- 本地上传流程仅提交当前文档实际引用的图片文件，避免无关图片参与上传请求。
- 同一批本地上传请求按文档拆分执行，降低单次请求失败对其他文档的影响。
- 批量任务执行中持续更新状态，减少长耗时任务期间用户重复点击或误刷新风险。

## 🔄 API 变更

新增接口：

- `POST /api/convert/local-batch`：批量上传 Markdown 中引用的本地图片并替换地址。

接口说明：

- 请求类型为 `multipart/form-data`。
- 表单字段 `manifest` 保存目标图床配置、Markdown 文档内容和本地图片路径映射。
- 图片文件通过 manifest 中的 `file_key` 与 multipart 文件字段关联。
- 远程 `http://`、`https://` 图片地址不会被本地上传流程替换。

现有接口保持兼容：

- `POST /api/convert/process` 继续用于单文档远程图片转换。
- `POST /api/convert/batch` 保留批量转换接口。
- `GET /api/convert/records` 继续返回转换历史记录。

## 📊 数据库变更

本版本无数据库 schema 变更。

转换历史继续写入 `conversion_records` 表；本地上传来源记录为 `source_picbed = local`。

## 🔄 升级说明

从 v1.0.0 升级到 v2.0.0 无需执行数据库迁移 SQL。

升级后建议注意：

- 如通过 Nginx 反向代理部署，请确保上传大小限制满足本地图片上传需求，例如 `client_max_body_size 50m` 或按实际场景调整。
- 本地上传依赖浏览器文件选择授权，后端不会直接读取用户电脑上的本地路径。
- 本地上传接口单张图片大小仍受后端 20MB 限制保护。
