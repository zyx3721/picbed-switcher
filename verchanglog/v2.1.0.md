## 📋 版本概述

v2.1.0 聚焦账号安全与密码找回体验。新增忘记密码功能，支持用户通过邮箱接收一次性重置链接并设置新密码；同时完善 SMTP 安全配置与 HTML 卡片式邮件模板，修复重置链接场景下的前端构建类型问题，确保发布版本可正常构建。

---

## ✨ 新增特性

- 新增登录页“忘记密码？”入口，支持用户通过邮箱申请密码重置链接。
- 新增密码重置页面状态，访问带 `reset_token` 的链接时自动进入重置密码流程。
- 新增 `/api/auth/password/forgot` 接口，校验邮箱存在后生成一次性重置令牌并发送邮件。
- 新增 `/api/auth/password/reset` 接口，支持通过未过期、未使用的重置令牌设置新密码。
- 新增 SMTP 邮件发送配置，支持发件人显示名、隐式 TLS、STARTTLS、明文 SMTP 和自动安全模式。
- 新增 HTML 卡片式密码重置邮件模板，并保留纯文本 fallback。

---

## 🐛 Bug 修复

- 修复前端工作区上下文未提供 `activatePasswordReset` 导致生产构建类型检查失败的问题。
- 优化邮件中的备用重置链接展示方式，减少邮箱客户端安全跳转前缀影响复制访问的问题。

---

## 🚀 性能优化

- 密码重置令牌仅保存 SHA-256 哈希，避免明文令牌落库。
- 申请新重置链接时自动废弃同一用户旧的未使用令牌，减少可用令牌数量。
- 密码重置令牌默认 30 分钟过期，并在使用后标记为已使用。

---

## 🔧 重构优化

- 认证表单扩展为登录、注册、找回密码和重置密码四种模式，复用现有提示与校验机制。
- 后端配置模块新增应用基础地址、密码重置令牌有效期和 SMTP 邮件配置项。
- 新增独立邮件发送工具，集中处理 SMTP 认证、TLS/STARTTLS 升级、发件人格式和 multipart 邮件内容构造。
- 登录页底部入口调整为左侧注册、右侧忘记密码，提升登录场景下的入口可见性。

## 🔄 API 变更

新增接口：

- `POST /api/auth/password/forgot`：发送密码重置邮件。
- `POST /api/auth/password/reset`：使用重置令牌设置新密码。

接口说明：

- `/api/auth/password/forgot` 请求体包含 `email`，后端会校验邮箱格式与用户邮箱是否存在。
- `/api/auth/password/reset` 请求体包含 `token`、`new_password`、`confirm_password`。
- 密码重置链接默认使用 `APP_BASE_URL` 拼接 `reset_token` 查询参数。
- 注册、登录、密码找回、密码重置和健康检查无需携带 JWT，其余接口仍需 `Authorization: Bearer <token>`。

现有接口保持兼容：

- `POST /api/auth/register` 继续用于用户注册。
- `POST /api/auth/login` 继续用于用户名或邮箱登录。
- `PUT /api/auth/password` 继续用于已登录用户修改当前密码。
- `PUT /api/auth/email` 继续用于已登录用户修改邮箱。

## 📊 数据库变更

新增数据表：

- `password_reset_tokens`：保存密码重置令牌哈希、用户 ID、过期时间、使用时间、请求 IP 和创建时间。

新增索引：

- `idx_password_reset_tokens_user_id`：按用户查询重置令牌。
- `idx_password_reset_tokens_expires_at`：按过期时间查询重置令牌。

## 🔄 升级说明

从 v2.0.0 升级到 v2.1.0 时，应用启动会通过 GORM AutoMigrate 自动创建 `password_reset_tokens` 表。

如需手动执行 SQL，请参考 `backend/migrations/001_init_schema.sql` 中的 `password_reset_tokens` 表结构和索引定义。

启用忘记密码功能前，请在后端或 Docker Compose 环境变量中配置：

```env
APP_BASE_URL=https://your-domain.com
PASSWORD_RESET_TOKEN_TTL_MINUTES=30
SMTP_HOST=smtp.example.com
SMTP_PORT=587
SMTP_SECURITY=auto
SMTP_USERNAME=noreply@example.com
SMTP_PASSWORD=your_smtp_password
SMTP_FROM=noreply@example.com
SMTP_FROM_NAME=PicBed Switcher
```

`SMTP_SECURITY` 可选值为 `auto`、`ssl`、`starttls`、`none`。465 端口通常使用 `ssl`，587 端口通常使用 `starttls`。
